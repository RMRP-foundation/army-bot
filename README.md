# Discord-бот управления фракцией Армии (Regular Army Bot)

Приватный Discord-бот, разработанный для автоматизации кадровых, логистических и административных процессов внутри виртуальной фракции. Бот работает в режиме Single-Guild и интегрирован в инфраструктуру проекта.

## 🌟 1. Описание функционала
*   **Кадровый документооборот:** Интерактивные UI-панели (Views/Modals) для подачи рапортов (ПСЖ, переводы, восстановления, отгулы, запросы ролей).
*   **Иерархия и доступ:** Автоматическая выдача/снятие ролей, изменение никнеймов в соответствии с должностью, подразделением и званием.
*   **Склад и логистика:** Запрос снабжения с подсчетом лимитов по категориям, кулдаунами (3 часа) и аудитом выдачи.
*   **Управление нарушениями:** Модуль Черного Списка (ЧС) с автоматическим расчетом неустойки (если отработано менее 5 дней).
*   **Ежедневные задачи:** Планировщик для отправки ежедневных сводок по расписанию.

## ⌨️ 2. Список команд
**Slash-команды (App Commands):**
*   `/blacklist [пользователь] [дни] [причина] [доказательства]` — Занесение в ЧС (Доступ: Капитан+).
*   `/unblacklist [пользователь] [причина]` — Вынос из ЧС (Доступ: Капитан+).
*   `/members [подразделение]` — Постраничный просмотр состава подразделения (Доступ: Капитан+).

**Контекстное меню (ПКМ по пользователю -> Приложения):**
*   `Отредактировать` — Изменение имени, статика, звания, подразделения и должности.
*   `Повысить (+1 зв.)` — Быстрое повышение на 1 порядковое звание.
*   `Уволить` — Увольнение с указанием причины (с автоматическим снятием ролей).

**Admin-команды (Префиксные, только для администраторов):**
*   `!refresh_roles`, `!refresh_timeoff`, `!refresh_dismissal`, `!refresh_supplies`, `!refresh_audit`, `!refresh_transfer`, `!refresh_reinstatement`, `!refresh_sso`, `!refresh_materials` — Обновление закрепленных UI-панелей в соответствующих каналах.

## 📚 3. Используемые библиотеки
*   `discord.py` — Базовый фреймворк для работы с Discord API (вкл. `app_commands`, `tasks`).
*   `beanie` — Асинхронный ODM для работы с MongoDB.
*   `motor` — Асинхронный драйвер MongoDB для Python.
*   `pydantic` — Валидация данных и схемы БД.
*   `python-dotenv` — Чтение переменных окружения.
*   `cachetools` — In-memory кэширование (оптимизация запросов).

## 🛡 4. Требуемые права (Permissions) и Интенты
Бот спроектирован с учетом принципа минимальных привилегий (Регламент п.4).

**Необходимые Discord Intents (Интенты):**
*   ✅ `Server Members Intent` — Требуется для синхронизации ролей, проверки наличия юзера на сервере и обновления никнеймов.
*   ✅ `Message Content Intent` — Требуется для обработки административных префиксных команд (`!refresh_...`).
*   ❌ *Privileged Presences Intent — НЕ требуется и отключен.*

**Необходимые права бота (Bot Permissions):**
*   `Manage Roles` (Управление ролями) — Выдача/снятие ролей званий и подразделений.
*   `Manage Nicknames` (Управление никнеймами) — Установка формата `[ВК] Р-й | Имя Фамилия`.
*   `Read Messages / View Channels` — Просмотр каналов с заявками.
*   `Send Messages / Send Messages in Threads` — Отправка логов и ответов.
*   `Manage Messages` — Удаление старых UI-сообщений при обновлении меню.
*   `Embed Links` & `Attach Files` — Формирование карточек аудита и заявок.

## 📊 5. Логирование и Аудит (согласно п. 7.3 Регламента)
Бот ведет двойное логирование всех критических действий:
1.  **Локальные логи (Системные):** Запись ошибок, предупреждений и состояния БД в файл `bot.log` и `stdout` (через стандартный модуль `logging`).
2.  **Discord-Аудит (Игровые и кадровые действия):** Все действия (принятие, увольнение, повышение, изменение должности/подразделения, выдача склада, смена ника) логируются в выделенные Discord-каналы (указываются в `config.py`) через модуль `utils/audit.py`. Логи включают инициатора, цель, время, причину и ссылки на доказательства.

## 🚀 6. Инструкция по развертыванию

### Шаг 1: Инфраструктура Discord
Для работы бота администрация должна создать следующие сущности (ID вносятся в `config.py`):
*   Каналы для логов: *Кадровый аудит, Аудит склада, Статик-лог, Блэклист.*
*   Каналы для заявок: *Запрос ролей, Увольнения, Восстановления, Отгулы, Переводы, Запрос склада.*

### Шаг 2: Установка зависимостей
```bash
git clone <url-репозитория>
cd <папка-проекта>
pip install -r requirements.txt
```

### Шаг 3: Переменные окружения
Создайте файл `.env` в корневой директории:
```env
# Обязательно к заполнению
TOKEN=ваш_токен_бота
ENVIRONMENT=production # production или development

# СУБД (MongoDB согласована с администрацией)
MONGO_URI=mongodb://localhost:27017
MONGO_DB_NAME=RegularArmyDB
```
*Токен хранится исключительно в `.env` (Требование п. 3.4).*

### Шаг 4: Запуск
```bash
python main.py
```
*Примечание: При запуске бот проверяет соединение с БД. При недоступности БД процесс будет аварийно завершен для предотвращения потери данных (Fail Fast).*
